# Bug-Report-Website-Testing
Баг-репорт - тестирование мобильной версии сайта.

## Описание проекта

Данный репозиторий содержит баг-репорт, составленный в процессе ручного тестирования мобильной версии веб-сайта.

Цель проекта:

* поиск дефектов в приложении
* оформление баг-репортов
* описание шагов воспроизведения ошибки
* фиксация фактического и ожидаемого результата
* использование скриншотов для подтверждения дефектов

## Область тестирования

Тестирование проводилось для мобильного интерфейса сайта.

Проверялись следующие функции:

* переход в каталог товаров
* работа личного кабинета
* переход по внешней ссылке (Telegram)
* скорость загрузки интерфейса

## Найденные дефекты

В процессе тестирования было обнаружено **3 дефекта**:

1. **Длительная загрузка приложения**
   После перехода в каталог товаров загрузка приложения занимает более 10 секунд.

2. **Не открывается личный кабинет**
   После нажатия на иконку профиля страница личного кабинета не открывается.

3. **Не открывается ссылка на Telegram**
   При нажатии на иконку Telegram появляется белый экран вместо перехода по ссылке.

## Содержимое баг-репорта

Файл **Bug-Report.xlsx** содержит следующие поля:

* Номер бага 
* Название бага
* Приоритет 
* Предусловия
* Шаги воспроизведения
* Фактический результат
* Ожидаемый результат
* Скриншот

## Использованные инструменты

* Google Sheets
* Google Chrome
* Инструменты разработчика (DevTools)

## Цель проекта

Цель данного проекта — продемонстрировать практические навыки ручного тестирования, а также умение структурированно оформлять баг-репорты и документировать найденные дефекты.
